# ADR-016: MIT Licensing And No Community/Pro Split During First Year

## Status

Accepted

## English

Decision: The project uses the MIT license and has no Community/Pro separation during the first year.

Rationale: The first year should prioritize trust, adoption, architecture validation and public API stability instead of commercial packaging complexity.

Consequences: Advanced features may still be modular packages or plugins, but they are not Community/Pro gated during the first year.

## Turkce

Karar: Proje MIT lisansi kullanir ve ilk yil Community/Pro ayrimi yapmaz.

Gerekce: Ilk yil ticari paketleme karmasasi yerine guven, benimsenme, mimari dogrulama ve public API stabilitesi onceliklidir.

Sonuc: Advanced feature'lar modular package veya plugin olabilir, fakat ilk yil Community/Pro gate uygulanmaz.
