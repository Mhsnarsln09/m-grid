# ADR-013: MVP Boundary

## Status

Accepted

## English

Decision: MVP is split into Contract MVP and Usable Grid MVP.

Rationale: Contract work and usable grid behavior have different risk profiles. Splitting them prevents scope creep.

Consequences: Contract MVP has no real grid rendering behavior. Usable Grid MVP includes Vue rendering, DOM/vanilla foundation, ARIA navigation, fixed virtualization, sorting, filtering, pagination, selection, priority columns and optional theme.

## Turkce

Karar: MVP, Contract MVP ve Usable Grid MVP olarak ikiye ayrilir.

Gerekce: Contract calismasi ve usable grid davranisi farkli risk profillerine sahiptir. Bu ayrim scope creep'i engeller.

Sonuc: Contract MVP real grid rendering behavior icermez. Usable Grid MVP Vue rendering, DOM/vanilla foundation, ARIA navigation, fixed virtualization, sorting, filtering, pagination, selection, priority columns ve optional theme icerir.
