# Codex Prompt: Tam Kapsamlı Grid Analizi

Aşağıdaki metni Codex proje kökünde çalıştır:

```text
$grid-package-analysis becerisini kullan.

docs/original-analysis-prompt.md dosyasındaki gereksinimlere göre bu repository için kapsamlı ürün ve teknik mimari analizi hazırla.

Önce repository durumunu incele. Ardından özel ajanları iki dalga hâlinde açıkça oluştur:

Dalga A:
- product_competitor_researcher
- grid_architect
- api_typescript_designer
- styling_theming_specialist
- responsive_mobile_specialist
- performance_virtualization_specialist

Dalga B:
- accessibility_specialist
- data_state_specialist
- quality_security_specialist
- dx_packaging_docs_specialist

Her dalgadaki ajanların tamamlanmasını bekle. Raporları birleştirerek docs/analysis-output-template.md yapısına uygun bir taslak oluştur. Sonra synthesis_reviewer ajanını oluştur; taslağı ve uzman raporlarını bağımsız olarak incelemesini iste. Geçerli bulguları uygula.

Kod yazma. Sonuçları mümkünse şu dosyalara kaydet:
- docs/architecture-analysis.md
- docs/mvp-acceptance-criteria.md
- docs/performance-benchmark-plan.md
- docs/responsive-test-matrix.md
- docs/roadmap.md
- docs/adr/README.md

Her teknik kararda karar, gerekçe, alternatifler, avantajlar, dezavantajlar, riskler, önerilen çözüm ve kabul kriterlerini yaz.

Özellikle CSS manipülasyon kolaylığı, düşük specificity, CSS custom properties, slot/class sistemi, unstyled kullanım, container-query tabanlı responsive davranış, tablet/mobil stratejileri, virtualization-accessibility çatışmaları ve TypeScript inference kalitesini ayrıntılı ele al.

Bittiğinde oluşturulan dosyaları, temel mimari kararları, MVP sınırını, çözülmemiş konuları ve önerilen ilk implementation slice'ını özetle.
```
